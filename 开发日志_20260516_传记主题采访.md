## 二、产品决策记录

### 2.1 右侧栏功能取舍

讨论后决定，第一版右侧栏不再承担“亲情关怀、紧急求助、分享给家人、传记准备进度”等横向功能。

原因：

- “紧急求助”会把产品预期拉向老人安全监护，带来责任边界问题。
- “亲情关怀”和“分享给家人”更像后续家庭协作能力，不是第一版核心闭环。
- “传记准备进度”过于结果导向，而当前更需要帮助老人开始讲述。

最终右侧栏定位调整为：

> 今天想聊哪个主题？让老人或陪护者选择一个人生主题，由 AI 围绕该主题进行采访。

### 2.2 第一版 10 个主题

最终确定的第一版主题如下：

1. 我的孩童时代
2. 我的父母和家
3. 求学时候的日子
4. 年轻时候的日子
5. 工作与生计
6. 爱情与婚姻
7. 家庭与子女
8. 人生的转折点
9. 难忘的年代
10. 留给家人的话

这些主题只是“采访入口”，不是最终成书章节标题。后续写作章节标题会根据实际素材自动提炼。

### 2.3 进度规则

每个主题的进度不是简单按聊天时长计算，而是按“素材丰富度”判断。

统一评分维度：

| 维度 | 分值 | 含义 |
|------|------|------|
| 基础事实 | 20 | 时间、地点、人物、身份关系是否清楚 |
| 具体事件 | 20 | 是否讲出真实发生过的故事 |
| 细节画面 | 20 | 是否有生活细节、场景、物件、声音等 |
| 情绪感受 | 20 | 是否表达开心、害怕、遗憾、自豪、想念等 |
| 人生影响 | 20 | 是否说明这段经历如何影响后来的人生 |

状态分桶：

| 进度 | 状态文案 |
|------|----------|
| 0% | 还没开始 |
| 1-34% | 刚刚开始 |
| 35-64% | 已有故事 |
| 65-84% | 继续补充 |
| 85%+ | 素材已丰富 |

重要产品原则：

> 达到 85% 后，只提示“这个主题素材已经很丰富”，不强制切换主题，老人仍然可以继续讲。

## 三、数据库与数据结构

### 3.1 新增集合

新增腾讯云 CloudBase 集合：

```text
topic_profiles
```

设计原则：

- 不改动旧集合结构。
- 不重命名旧字段。
- 不改变 `memory_profiles` 的语义。
- 主题采访进度独立存储，避免影响已有自传生成链路。

### 3.2 `topic_profiles` 文档结构

每个用户对应一条主题档案：

```js
{
  userId: string,
  currentTopicId: "childhood",
  personProfile: {},
  topics: [
    {
      id: "childhood",
      title: "我的孩童时代",
      progress: 0,
      status: "not_started",
      summary: "",
      knownFacts: [],
      concreteStories: [],
      missingInfo: [],
      suggestedNextQuestion: "",
      lastDiscussedAt: null
    }
  ],
  allRichPromptShown: false,
  createdAt,
  updatedAt
}
```

### 3.3 当前已完成的数据能力

后端已支持：

- 登录/注册后自动获取或创建 `topic_profiles`。
- 当前主题默认是 `childhood`。
- 可通过 HTTP API 获取用户主题档案。
- 可通过 HTTP API 或 WebSocket 切换当前主题。
- 如果数据库中已有旧版本主题档案，后端会补齐缺失字段。

### 4.3 前端兜底修复

问题：

右侧标题已显示，但按钮没有出现。

根因：

按钮渲染依赖 `topicProfile?.topics`，而本地 `localhost:8000` 后端未连接或接口未返回时，`topicProfile` 为空，导致按钮列表为空。

修复：

新增：

```text
lovable_ui/src/lib/biographyTopics.ts
```

提供前端默认主题档案：

```ts
createFallbackTopicProfile(userId)
```

现在即使后端暂时不可用，右侧也会先显示 10 个主题按钮；后端返回后再覆盖为数据库中的永久状态。

## 七、当前未完成事项

下一阶段计划，但尚未实现：

1. **AI 对话提示词接入当前主题**
   - 混元回复时读取 `session.currentTopicId`。
   - 将当前主题、历史摘要、缺失信息拼入 prompt。

2. **DeepSeek 会话摘要更新主题进度**
   - 会话结束后输出 `topicAnalysis`。
   - 更新 `topic_profiles.topics[].progress`、`summary`、`missingInfo` 等字段。

3. **85% 提示机制**
   - 当当前主题达到 85% 后，AI 温柔提示“素材已丰富”，但不强制切换。

4. **全部主题丰富后的成书提示**
   - 当全部 10 个主题都达到 85% 后，弹窗提示“可以书写关于我的故事”。

5. **基础档案自然采集**
   - 性别、籍贯、民族、年龄等信息采用“自然问，可跳过”的方式采集。

## 八、当前产品状态

当前版本已经具备：

- 右侧主题选择 UI。
- 默认主题显示。
- 当前主题高亮。
- 主题进度条展示。
- 主题状态文案。
- 当前主题可持久化保存到腾讯云 `topic_profiles`。

当前版本尚未具备：

- AI 真正根据当前主题改变采访策略。
- 主题进度根据对话内容自动增长。
- 主题历史摘要驱动下一句追问。

因此当前阶段可定义为：

> 主题采访 UI 与持久化底座完成，下一步进入 AI 主题上下文与素材进度分析。
