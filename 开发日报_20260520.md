## 一、模型与语音供应商切换方案落地

- 保留原有混元主对话能力。
- 保留原有腾讯 ASR/TTS 能力。
- 保留 DeepSeek 摘要、自传生成、主题分析链路，不做替换。
- 新增 `LLM_PROVIDER` 配置，用于选择主对话供应商。
- 新增 `VOICE_PROVIDER` 配置，用于选择语音供应商。
- 当前服务器配置为：

```env
LLM_PROVIDER=doubao
VOICE_PROVIDER=doubao
```

- 后续如果要切回混元/腾讯，只需要调整环境变量，不需要改业务代码。

## 三、豆包语音能力接入

新增豆包语音链路，用于替换腾讯语音能力，但不删除腾讯实现。

### 3.1 豆包录音文件识别 2.0

- 用于替换腾讯 ASR。
- 当前前端是“录完一段再识别”的交互，因此采用录音文件识别，而不是实时语音交互。
- 使用资源 ID：

```env
DOUBAO_ASR_RESOURCE_ID=volc.bigasr.auc_turbo
```

- 兼容前端原有 PCM 音频输入，后端转换为豆包接口可识别的音频格式。

### 3.2 豆包语音合成 2.0

- 用于替换腾讯 TTS。
- 输出格式继续使用 `mp3`，避免改动前端 `decodeAudioData` 播放逻辑。
- 当前配置：

```env
DOUBAO_TTS_RESOURCE_ID=seed-tts-2.0
DOUBAO_TTS_VOICE=zh_female_vv_uranus_bigtts
DOUBAO_TTS_FORMAT=mp3
DOUBAO_TTS_SAMPLE_RATE=16000
```

## 五、登录与数据统计问题修复

### 5.1 用户端登录卡在处理中

- 排查发现根因是后端进程因启动日志变量错误崩溃，导致前端登录请求无法完成。
- 修复后端启动错误。
- 同时增强前端登录失败兜底：
  - 请求失败会显示错误提示。
  - loading 状态会正常恢复。
  - 避免按钮一直停留在“处理中”。

### 5.2 管理员端最新聊天记录不显示

- 排查发现管理员详情页对话记录按旧时间升序查询，数据量超过默认分页后，新记录被截断。
- 调整查询逻辑为按 `timestamp` 倒序获取最新记录。
- 管理员端现在可以看到最新对话。

### 5.3 总对话轮数统计不一致

- 排查发现总览卡片使用 `.get().data.length` 统计，受数据库分页上限影响，只能拿到部分数据。
- 改为使用数据库 `.count()` 统计。
- 用户列表与总览统计口径保持一致。

## 八、前端 SSR 部署问题排查

部署过程中确认 `lovable_ui` 不是纯静态 SPA，而是 TanStack Start SSR 应用。

关键结论：

- `dist/client` 没有 `index.html` 是正常现象。
- Nginx 不能直接把 `dist/client` 当静态站点根目录。
- 正确方式是：
  - 启动前端 SSR preview 服务在 `127.0.0.1:3000`
  - Nginx 将 `/` 代理到前端 SSR 服务

同时发现构建产物与 preview 入口存在差异：

```text
实际产物：dist/server/index.js
preview 期望：dist/server/server.js
```

临时处理方式：

```bash
ln -sf index.js dist/server/server.js
```

后续如有时间，可将该步骤沉淀为正式 `postbuild` 脚本。

## 十一、当前限制与后续计划

### 11.1 公网 IP + HTTP 的限制

当前可以用于基础内测：

- 登录。
- 管理端。
- 文字聊天。
- AI 回复。
- 数据写入。
- TTS 播放。

但浏览器麦克风录音在公网 HTTP 下可能被限制。后续正式语音内测建议升级为：

```text
域名 + HTTPS + wss
```
