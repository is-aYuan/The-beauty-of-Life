# 代码变更提交与云服务器启动服务说明

本文档用于记录以后项目代码发生变化时，如何从本地提交到 GitHub，再让云服务器拉取最新代码、构建前端、重启服务。

当前项目信息：

```text
GitHub 仓库: https://github.com/is-aYuan/The-beauty-of-Life.git
服务器公网 IP: 124.174.21.48
服务器项目目录: /var/www/traceoflife
后端 PM2 服务: traceoflife-api
前端 PM2 服务: traceoflife-web
后端端口: 8000
前端 SSR 端口: 3000
Nginx 访问端口: 80
```

## 1. 总体流程

每次代码更新按这个顺序走：

```text
本地修改代码
  -> 本地测试
  -> git add 指定文件
  -> git commit
  -> git push origin main
  -> SSH 登录服务器
  -> git pull
  -> npm install
  -> 前端 npm run build
  -> 重启 PM2 服务
  -> 浏览器测试公网 IP
```

不要直接在服务器上改业务代码。服务器只负责运行和部署，代码源头以本地 GitHub 为准。

## 2. 本地提交代码

在 Mac 本地进入项目目录：

```bash
cd "/Users/ayuan/Documents/cc- The beauty of Life"
```

查看改动：

```bash
git status
```

查看具体差异：

```bash
git diff
```

只添加本次需要提交的文件。不要直接 `git add .`，避免把 `.env`、构建产物、临时文件误提交。

示例：

```bash
git add server.js \
  lib/providerConfig.js \
  lovable_ui/src/hooks/useStoryEngine.ts \
  docs/provider-switching.md
```

提交：

```bash
git commit -m "描述这次更新"
```

推送到 GitHub：

```bash
git push origin main
```

确认本地已经干净：

```bash
git status
```

理想输出：

```text
Your branch is up to date with 'origin/main'.
nothing to commit, working tree clean
```

## 3. 哪些文件不要提交

这些文件通常不要提交到 GitHub：

```text
.env
lovable_ui/.env.production
node_modules/
lovable_ui/node_modules/
dist/
lovable_ui/dist/
.DS_Store
.claude/settings.local.json
data/records/
```

其中 `.env` 和 `lovable_ui/.env.production` 只保留在服务器上。

如果不确定某个文件该不该提交，先执行：

```bash
git status
```

确认后再 `git add`。

## 9. 如果服务器 git status 不干净

服务器上如果看到类似：

```text
Changes not staged for commit
Untracked files
```

先不要 `git pull`。

优先判断是不是这些服务器本地文件：

```text
.env
lovable_ui/.env.production
node_modules/
lovable_ui/node_modules/
lovable_ui/dist/
.claude/settings.local.json
```

如果只是服务器环境文件，应该备份或忽略，不应该提交。

可以先做一个服务器本地 stash：

```bash
cd /var/www/traceoflife
git stash push -m "server local changes before deploy"
```

如果有未跟踪文件导致 pull 冲突，可以先备份到 `/root/traceoflife-backup`：

```bash
mkdir -p /root/traceoflife-backup
mv lovable_ui/.env.production /root/traceoflife-backup/.env.production 2>/dev/null || true
mv .claude/settings.local.json /root/traceoflife-backup/settings.local.json 2>/dev/null || true
```

然后再拉取：

```bash
git pull --ff-only origin main
```

拉取后把环境文件放回：

```bash
cp /root/traceoflife-backup/.env.production /var/www/traceoflife/lovable_ui/.env.production 2>/dev/null || true
```

如果不确定哪些文件可以动，先把 `git status` 输出保存下来，不要继续执行危险命令。

## 11. 启动服务

如果 PM2 服务已经存在，只是被停止了：

```bash
pm2 start traceoflife-api
pm2 start traceoflife-web
```

如果 PM2 里没有服务，需要重新创建：

```bash
cd /var/www/traceoflife
pm2 start server.js --name traceoflife-api

cd /var/www/traceoflife/lovable_ui
pm2 start node_modules/vite/bin/vite.js --name traceoflife-web -- preview --host 127.0.0.1 --port 3000

pm2 save
```

确认：

```bash
pm2 status
```

## 12. 停止服务

临时停止项目：

```bash
pm2 stop traceoflife-api
pm2 stop traceoflife-web
```

恢复：

```bash
pm2 start traceoflife-api
pm2 start traceoflife-web
```

彻底从 PM2 移除：

```bash
pm2 delete traceoflife-api
pm2 delete traceoflife-web
pm2 save
```

让公网 IP 直接打不开：

```bash
systemctl stop nginx
```

恢复 Nginx：

```bash
systemctl start nginx
```

## 14. 常见问题

### 14.1 git pull 失败

先看：

```bash
git status
```

常见原因：

```text
1. 服务器有未提交改动
2. 服务器有未跟踪文件和 GitHub 新文件冲突
3. 本地还没有 push 到 GitHub
```

处理原则：

```text
先保护服务器环境文件，再 pull。
不要直接 reset 或 restore 整个项目。
```

### 14.2 前端 500

看前端日志：

```bash
pm2 logs traceoflife-web --lines 80
```

如果出现：

```text
Cannot find module .../dist/server/server.js
```

执行：

```bash
cd /var/www/traceoflife/lovable_ui
ln -sf index.js dist/server/server.js
pm2 restart traceoflife-web
```

### 14.3 页面打开但接口失败

检查后端：

```bash
pm2 status
curl http://127.0.0.1:8000/health
pm2 logs traceoflife-api --lines 80
```

检查前端配置：

```bash
cat /var/www/traceoflife/lovable_ui/.env.production
```

应包含：

```env
VITE_API_BASE=http://124.174.21.48
VITE_WS_URL=ws://124.174.21.48/ws/chat
```

### 14.4 Nginx 配置改完没生效

执行：

```bash
nginx -t
systemctl reload nginx
```

如果 `nginx -t` 报错，按报错行号修配置，不要 reload。

## 15. 回滚到上一个版本

如果刚部署后发现严重问题，优先在 GitHub 找到上一个正常 commit。

服务器查看最近提交：

```bash
cd /var/www/traceoflife
git log --oneline -5
```

临时回到某个提交：

```bash
git checkout 旧commitID
```

然后重新安装、构建、重启：

```bash
npm install
pm2 restart traceoflife-api

cd /var/www/traceoflife/lovable_ui
npm install
npm run build
ln -sf index.js dist/server/server.js
pm2 restart traceoflife-web
```

确认没问题后，后续应该在本地 main 分支修复并重新 push，而不是长期让服务器停留在 detached HEAD 状态。

恢复到 main 最新：

```bash
cd /var/www/traceoflife
git checkout main
git pull --ff-only origin main
```

## 16. 后续换域名时的变化

买域名并配置 HTTPS 后，前端 `.env.production` 改为：

```env
VITE_API_BASE=https://你的域名
VITE_WS_URL=wss://你的域名/ws/chat
```

然后重新构建前端：

```bash
cd /var/www/traceoflife/lovable_ui
npm run build
ln -sf index.js dist/server/server.js
pm2 restart traceoflife-web
```

Nginx 的 `server_name` 从 `_` 改成你的域名，再用 Certbot 配 HTTPS。
