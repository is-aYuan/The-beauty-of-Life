## 一、移动端产品方案确认

围绕手机端主界面拥挤的问题，确认移动端采用与桌面端不同的工程化布局，而不是强行压缩桌面三栏结构。

核心设计方向：

- 桌面端继续保留现有左侧导航、中间对话、右侧主题栏布局。
- 移动端单独启用移动适配布局。
- 顶部采用模块切换导航：
  - 对话
  - 回忆库
  - 家庭
  - 设置
- 主界面明确区分：
  - 主题区
  - 录音控制区
  - 内容展示区
- 原草图中的“顶部 4 个模块”和底部功能区，后续均按可上下伸缩的移动端模块设计。
- 移动端避免把多个桌面区域横向挤压到同一屏，改为按模块切换和纵向展开。

## 二、移动端体验问题梳理

今日基于真机访问截图，梳理了当前移动端已经适配后的四类问题：

### 2.1 退出登录入口缺失

- 移动端顶部下拉区域原本功能较多，观感偏复杂。
- 决定移除多余下拉展示功能。
- 退出登录按钮后续放入顶部下拉区域，作为移动端账号操作入口。

### 2.2 设置页最大字体后无法继续下滑

- 问题表现：字体调到最大后，设置页底部按钮和滑动条不可达。
- 判断根因是移动端容器高度、滚动容器和字体缩放未完全解耦。
- 后续方案：
  - 设置页使用独立纵向滚动容器。
  - 固定顶部导航不参与内容高度计算。
  - 字体调节区保留足够底部安全间距。

### 2.3 移动端长按说话无效

- 真机点击“长按说话”后，后端日志出现：

```text
收到指令: user_speech_ended
音频数据太短，忽略
```

- 判断存在两个层面原因：
  - 移动端触摸事件和桌面鼠标事件处理不完全一致，可能一按下就触发结束。
  - 公网 HTTP/IP 访问不属于安全上下文，浏览器不会授予麦克风权限。

### 2.4 “放桌上畅聊”按钮无实际动作

- 点击后只出现抖动，说明前端有交互反馈，但实际语音会话链路未进入有效状态。
- 后续需要将该模式的入口、权限检查、WebSocket 状态、录音启动状态串联起来。

## 三、公网 HTTP 与麦克风权限边界确认

今日重点确认了移动端语音权限无法弹出的根因。

关键结论：

- `localhost` 可以请求麦克风权限。
- 局域网 IP 或公网 HTTP/IP 在大多数移动浏览器中不属于安全上下文。
- `http://120.79.128.111` 可以访问页面，但不能稳定触发麦克风授权。
- 移动端语音能力正式可用，需要使用 HTTPS 域名访问。

因此确定后续正式访问链路为：

```text
https://gushifang.cn
```

而不是长期使用公网 IP。

## 八、SSR 白屏与多次刷新问题修复

线上访问 `/` 或 `/login` 时，曾出现需要刷新多次页面才出现、偶发白屏的问题。

排查后发现服务端日志存在：

```text
ReferenceError: localStorage is not defined
```

根因：

- 当前前端是 TanStack Start SSR 应用。
- SSR 阶段运行在 Node.js 中，没有浏览器环境的 `localStorage`。
- 首页入口中直接读取 `localStorage`，导致 SSR 渲染失败。

修复方案：

- 在首页登录态判断处增加浏览器环境判断。
- 只在客户端存在 `localStorage` 时读取本地登录态。
- 新增 SSR 安全测试，防止后续再次引入同类问题。

相关文件：

```text

tests/ssrSafety.test.js
```

## 九、WebSocket 反复重连问题修复

线上页面曾出现提示：

```text
爷爷，网络打了个盹，正在努力重连...
```

后端日志表现为大量连续：

```text
[连接] 客户端 ... 已连接
[断开] 客户端 ...
```

排查判断：

- 前端在清理旧 WebSocket 时，会触发 `onclose`。
- `onclose` 没有区分“主动关闭”和“异常断线”。
- 旧连接被主动关闭后仍然触发自动重连，形成连接/断开循环。

修复方案：

- 新增主动关闭标记。
- 新增重连定时器清理逻辑。
- 新增当前 socket 身份校验，避免旧 socket 的回调影响新连接。
- 用户退出或组件卸载时，按主动关闭处理，不再进入异常重连。

相关文件：

```text
lovable_ui/src/hooks/useStoryEngine.ts
tests/websocketLifecycle.test.js
```
